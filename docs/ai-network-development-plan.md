# AI Native Ad Network 开发计划（主文档）

- 文档版本：v0.2
- 最近更新：2026-02-17
- 维护方式：此文档作为项目长期路线图，后续所有阶段调整和优先级变更均在此增量更新。

## 0. 远期目标
打造一个 **AI 原生的 AI Network**：先从应用侧广告聚合平台切入，逐步完成资源整合、结算与竞价能力，最终形成面向 AI Native App 的广告基础设施。

## 1. 核心项目：基础广告位管理系统（第一阶段）
当前最核心任务是开发面向 AI 应用的 SDK。

### 1.1 开发者集成
- 支持开发者通过 SDK 集成不同广告位。
- 提供标准化接入方式，降低集成成本。

### 1.2 广告拉取
- 广告位资源由 Network 从外部资源池拉取。
- 现阶段不自研模型，优先对接第三方 DSP / 广告联盟。
- 同时支持接入自有库存产品。

### 1.3 配置管理
- 支持按广告位配置展示类型。
- 支持配置触发时机。
- 支持配置参数（可扩展字段）。

### 1.4 智能触发
- 通过高层参数管理广告出现逻辑。
- 代表参数：预期收益金额、用户 Intent 程度等。

### 1.5 开发者后台（Developer Dashboard）
- 提供接入方可操作的 API Dashboard（收益 + 设置 + 决策日志）。
- 支持租户管理员自定义参数策略（Placement 与 Trigger）。
- 保留平台级 guardrail（参数上下限、安全话题限制、审计日志）。

## 2. 资源对接与整合（第二阶段）
本阶段重点是广告资源整合能力。

### 2.1 联盟接入
- 接入多种广告联盟。
- 建立类似 Max / MoPub 的资源池管理系统。

### 2.2 智能匹配
- 基于广告主提供素材，自动匹配最合适广告位。

### 2.3 结算系统
- 建立基础报价体系。
- 建立简化结算体系。

## 3. 竞价系统（第三阶段）
在具备足够报价方与出价方后，构建完整竞价系统。

- 当前状态：占位阶段，细节待后续设计。
- 后续需明确：竞价机制、排序逻辑、风控规则、实时性要求。

## 4. 模拟与测试模块：AI Native App 容器
用于调控与实验 SDK 效果，形成可验证环境。

### 4.1 逻辑用途
- 通过实验体验观察不同参数（如 Mix 参数）下的展示状态。

### 4.2 内容形式
- 内置行业通用 AI Native App 形态：Agent、Chatbot、其他 Client 类型。

### 4.3 首个应用
- 首先开发一个简单 AI Chatbot 作为第一个 App。

### 4.4 管理后台容器
- 新增 Developer Dashboard 模拟器，作为接入方运营与配置入口。
- 先完成 4 个基础页面：Overview、Placements、Triggers、Decision Logs。
- 后续与广告聚合平台 API 联调，替换本地 mock 数据。
