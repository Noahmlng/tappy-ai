# AI Native Ad Network 开发计划（主文档）

- 文档版本：v0.6
- 最近更新：2026-02-21
- 维护方式：本文件用于统一阶段目标与战略焦点；保持概念层表达，不展开字段级执行细节。

## 0. 总体方向

1. 长期目标不变：打造新的 AI Native Network。
2. 近期策略明确：先把 Mediation 做深做强，再升级 SSP，同时持续观察 DSP 机会。
3. 方法论：先建立可快速接入的标准与能力，再通过数据与认知积累形成差异化壁垒。

## 1. 第一阶段：Mediation First（当前核心）

### 1.1 目标对齐

1. 让 AI 应用以低成本、低摩擦方式接入广告网络。
2. 形成稳定、可复制的快速接入路径。

### 1.2 核心 Focus

1. 完成第一阶段战略目标：定义“快速接入标准”。
2. 在兼容现有生态的基础上，持续提升接入质量与效率。

### 1.3 数据挖掘

1. 持续挖掘能提升 DSP 出价意愿的数据内容与洞察。
2. 无论是新增数据维度还是新的 insights，目标都是帮助 Ads Network 与广告主发现更优广告机会。

### 1.4 认知积累

1. 持续定义 AI Workflow 中可沉淀、可复用、可进化的数据认知。
2. 把“认知积累”作为长期竞争力来源，而不是一次性项目。

### 1.5 当前两步执行（概念层）

1. 第一步：与 SSP 输入标准对齐，保证主流生态可接入。
2. 第二步：增加 Agency 库或阶段性接入，扩展可用供给与验证范围。

## 2. 动态跟进新应用（并行能力）

1. 快速响应：针对未来新类型应用（例如 OpenCloud）保持动态、快速跟进。
2. 高兼容性：当新平台出现时，提供可直接接入的兼容广告形式。

## 3. Agent 时代的广告形式（战略押注）

1. 入口迁移：押注 Agent 将成为重要工作入口，以及 Agent to Agent 的追踪入口。
2. 执行转移：大量任务由 Agent 执行，人类注意力停留时间可能显著缩短。
3. 触达逻辑：广告触达既要面向 Agent，也要面向人类在 AI 应用链路内的关键节点。
4. 形式定义：不再单纯依赖传统注意力经济，而是按应用形态定义更有效的广告形式。

## 4. 第二阶段：升级为 SSP

目标：
1. 从 Mediation 升级为 SSP，并将其作为 Market Object。

核心能力：
1. Traffic 质量判断。
2. View Opportunity 分析。
3. 用户 Intent 识别。
4. 用户建模（User Modeling）。

预期结果：
1. 形成品牌信誉更强的 SSP 信源。
2. 给 DSP 提供风险更低、质量更高的 Bid Request。

## 5. 第三阶段：探索自建 DSP

1. 当前结论：保持开放，不提前下定论。
2. 策略：且看且走，持续观察我们对 DSP 的真实双向影响。
3. 原则：只有当价值被明确验证时，才推进自建 DSP 的正式建设。

## 6. By-Step 执行路线（Milestones + Substeps）

### Stage 1：Mediation 冷启动阶段（从 0 到 1）

阶段现实：
1. 初期应用方与广告商规模有限。
2. 受限于规模与连接条件，早期难以直接接入 DSP 与大型 Ads Network。

当前实现策略：
1. 通过多广告联盟接入 + 模拟广告库，构建可运行的 Ads Network 基础能力。
2. 将研发重点集中在 Mediation 层，同时补齐实际接入所需的基础开发。

主要动作（Substeps）：
1. 建立“可接入、可验证、可回退”的接入标准框架。
2. 建立基础供给池（联盟 + 模拟库）并打通基本链路。
3. 建立初步质量反馈机制，为后续阶段提供数据基础。

里程碑（Milestone）：
1. 能稳定支撑首批 AI 应用低摩擦接入。
2. 能持续输出可用广告机会并形成基础价值证明。

---

### Stage 2：Mediation 扩展阶段（从 1 到 N）

阶段触发：
1. 已开始接入部分 Ads Network。

阶段重点：
1. 大幅投入不同类型 placement 的能力建设。
2. 让不同类型的广告应用都能接入，并在过程中持续挖掘价值。

主要动作（Substeps）：
1. 扩展并分层管理 placement 形态（按应用类型与链路场景）。
2. 建立“不同应用类型 -> 不同广告机会”的方法体系。
3. 持续优化接入效率与变现效果之间的平衡。

里程碑（Milestone）：
1. 形成可规模化复用的 placement 能力矩阵。
2. 在更多应用类型中验证 Mediation 的增量价值。

---

### Stage 3：SSP 化阶段（从 Mediation 到 Market Object）

阶段触发：
1. 拥有足够流量与稳定需求基础。

阶段重点：
1. 不再只是“接入网络”，而是建设自己的 SSP。
2. 建立可持续迭代的流量标准，并接入 DSP 交易体系。

主要动作（Substeps）：
1. 将流量质量、View Opportunity、Intent、User Modeling 能力产品化。
2. 输出可被 DSP 消费与信任的标准化信号。
3. 在接入与交易过程中持续迭代流量标准。

里程碑（Milestone）：
1. SSP 作为独立 Market Object 被稳定消费。
2. 对 DSP 输出“风险更低、质量更高”的 Bid Request 形成稳定认知。

---

### Stage 4：DSP 探索阶段（长期可选）

阶段触发：
1. 上述阶段验证出明确的网络与交易优势。

阶段重点：
1. 在具备条件时探索自建 DSP 的可行性。

主要动作（Substeps）：
1. 评估自建 DSP 的战略价值与投入产出。
2. 通过阶段性实验验证是否存在显著增量空间。

里程碑（Milestone）：
1. 形成“继续强化 SSP”或“进入 DSP 建设”的明确决策。

## 7. 设计与推进原则

1. Overall development plan 保持目标对齐与概念清晰，不下钻到过多执行细节。
2. 标识重点是必要的，但不必在本文件展开到字段命名或实现层。
3. 低优先级模块可用简化实现先跑通。
4. 核心模块必须先明确逻辑定义，再进入工程化落地。
