# AI Native Ad Network 开发计划（主文档）

- 文档版本：v0.5
- 最近更新：2026-02-21
- 维护方式：本文件用于统一阶段目标与战略焦点；保持概念层表达，不展开字段级执行细节。

## 0. 总体方向

1. 长期目标不变：打造新的 AI Native Network。
2. 近期策略明确：先把 Mediation 做深做强，再升级 SSP，同时持续观察 DSP 机会。
3. 方法论：先建立可快速接入的标准与能力，再通过数据与认知积累形成差异化壁垒。

## 1. 第一阶段：Mediation First（当前核心）

### 1.1 目标对齐

1. 让 AI 应用以低成本、低摩擦方式接入广告网络。
2. 形成稳定、可复制的快速接入路径。

### 1.2 核心 Focus

1. 完成第一阶段战略目标：定义“快速接入标准”。
2. 在兼容现有生态的基础上，持续提升接入质量与效率。

### 1.3 数据挖掘

1. 持续挖掘能提升 DSP 出价意愿的数据内容与洞察。
2. 无论是新增数据维度还是新的 insights，目标都是帮助 Ads Network 与广告主发现更优广告机会。

### 1.4 认知积累

1. 持续定义 AI Workflow 中可沉淀、可复用、可进化的数据认知。
2. 把“认知积累”作为长期竞争力来源，而不是一次性项目。

### 1.5 当前两步执行（概念层）

1. 第一步：与 SSP 输入标准对齐，保证主流生态可接入。
2. 第二步：增加 Agency 库或阶段性接入，扩展可用供给与验证范围。

## 2. 动态跟进新应用（并行能力）

1. 快速响应：针对未来新类型应用（例如 OpenCloud）保持动态、快速跟进。
2. 高兼容性：当新平台出现时，提供可直接接入的兼容广告形式。

## 3. Agent 时代的广告形式（战略押注）

1. 入口迁移：押注 Agent 将成为重要工作入口，以及 Agent to Agent 的追踪入口。
2. 执行转移：大量任务由 Agent 执行，人类注意力停留时间可能显著缩短。
3. 触达逻辑：广告触达既要面向 Agent，也要面向人类在 AI 应用链路内的关键节点。
4. 形式定义：不再单纯依赖传统注意力经济，而是按应用形态定义更有效的广告形式。

## 4. 第二阶段：升级为 SSP

目标：
1. 从 Mediation 升级为 SSP，并将其作为 Market Object。

核心能力：
1. Traffic 质量判断。
2. View Opportunity 分析。
3. 用户 Intent 识别。
4. 用户建模（User Modeling）。

预期结果：
1. 形成品牌信誉更强的 SSP 信源。
2. 给 DSP 提供风险更低、质量更高的 Bid Request。

## 5. 第三阶段：探索自建 DSP

1. 当前结论：保持开放，不提前下定论。
2. 策略：且看且走，持续观察我们对 DSP 的真实双向影响。
3. 原则：只有当价值被明确验证时，才推进自建 DSP 的正式建设。

## 6. 设计与推进原则

1. Overall development plan 保持目标对齐与概念清晰，不下钻到过多执行细节。
2. 标识重点是必要的，但不必在本文件展开到字段命名或实现层。
3. 低优先级模块可用简化实现先跑通。
4. 核心模块必须先明确逻辑定义，再进入工程化落地。
